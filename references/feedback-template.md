# OKR 穿透对齐 - 通信与反馈模板

## 第一轮：快速分配消息模板

向每个成员发送 `sessions_send` 时，使用以下 prompt 结构：

```
【OKR 穿透对齐 - 快速分配】

## 继承的 Objective
{O1 文本}

## 你负责的 Key Results
{该成员的 KR 列表}

## 业务背景
{相关业务背景说明}

## 请快速反馈
请针对分配给你的 KR，快速反馈以下内容（JSON格式）：

```json
{
  "confidence": 0-10,
  "my_krs": "你负责的 KR 列表",
  "help_needed": "需要我（总监）提供什么支持",
  "risks": "你看到的潜在风险或不确定性"
}
```

注意：暂时不需要检查工具和 SKILL，快速回复即可。
请严格按此 JSON 格式回复，不要附加其他格式。
```

## 第一轮反馈 JSON Schema

```json
{
  "confidence": "整数 0-10，10 为最高信心",
  "my_krs": "字符串，你负责的 KR 列表",
  "help_needed": "字符串，需要的帮助和支持",
  "risks": "字符串，潜在风险或不确定性"
}
```

## 第二轮：深度检查消息模板（可选）

如果用户要求深度检查工具和 SKILL：

```
【OKR 穿透对齐 - 深度能力检查】

基于你负责的 KR，请执行以下能力自检：

1. **查阅自己的 skill 列表**：检查你有哪些 skill 可以用于完成以上 KR
2. **查阅可用的 MCP 工具列表**：检查你有权限使用的 MCP 工具
3. **识别可用工具**：对于每个 KR，列出你能使用的具体 skill 或 MCP 工具
4. **禁止试运行**：不要执行任何实际操作，只做能力识别
5. **记录到 memory**：把发现的工具能力写入你的 MEMORY.md

请补充反馈（在原有 JSON 基础上添加）：

```json
{
  "available_tools": ["Skill: xxx", "MCP: xxx", "其他工具: xxx"],
  "tool_check_status": "已完成 / 进行中 / 遇到问题"
}
```
```

## 确认消息模板

```
【OKR 穿透对齐 - 最终确认】

根据所有成员的反馈，综合后的风险和需求如下：

{风险和需求汇总（按优先级分类）}

最终确认你负责的 OKR：

## Objective
{O1 文本}

## 你的 Key Results（最终）
{最终 KR 列表}

请确认是否同意。如同意，回复 "确认"。如有异议，请说明。
```
